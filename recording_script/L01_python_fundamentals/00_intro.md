Welcome to our first module.

Suppose that you receive a dataset with 1 year of daily observations on the revenue from a single store and that you are asked to make a forecast of the next value? The truth is that humans are pretty good at reasoning about small amounts of data and you could probably come up with a reasonable prediction...

Well, now imagine that you have 5 years of daily revenue data for 5,000 different stores and you're asked to make a prediction for each of the 5,000 stores -- This becomes a different scale of task entirely and becomes much more difficult to reason about. However, if we could encode the type of analysis that our brain was doing into a computer, then the computer should scale up to that number of observations more effectively.

This type of forecasting is just one example. A lot of economics work involves getting data into a usable state before any analysis can begin -- cleaning messy datasets, combining records from multiple sources, or preparing variables for a regression. These tasks are tedious or outright impossible to do by hand at scale, but they become manageable once you can write code to do them.

In this class, you will learn about how you can use a computer to improve your ability to do economics and generally how to manage data that is beyond the scale of what you can do in your mind or even in Microsoft Excel.

However, in order to use computers to do economics, we need know and understand the language that they use for communication -- This is known as a "programming language".

In this class, we will focus on the Python programming language for three reasons:

1. It is incredibly easy to read and write which allows us to move quickly from writing code to doing "data and economics"
2. It is becoming the lingua franca of science in many domains which means that it is broadly useful
3. A number of incredibly powerful tools have been developed that make Python a clear winner for data analysis, manipulation, and visualization which are all things that we care about as economists.

Depending on whether you are coming into this course with previous experience in programming, this first module will feel either _very fast_ or _very slow_.

If it feels slow, feel free to move quickly through this material! This module is meant as an opportunity to help get everyone in the class on the same page.

If it feels very fast, please take your time to make sure you understand and practice the basics! Programming is a lot like learning a game or a sport in the sense that it requires hands-on practice to develop the skills to play the game.
